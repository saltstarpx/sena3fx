# やがみプライスアクション ロジック抽出メモ

## 出典
- ローソク足の本｜やがみ (2020年4月)
- ローソク足の本2｜やがみ
- ポジり方の本｜やがみ
- 養分救済の本｜やがみ

---

## 1. レジサポの読み方

- ローソク足の「ヒゲ先」「ヒゲと実体の間（始値・終値）」に水平線を引く
- 反発回数が多いほど強固なレジサポ
- 自分の取引時間軸の約12倍の上位足でレジサポを確認（例：5分足なら1hで確認）
- スキャルピング〜デイトレは4h・日足でレジサポを確認

## 2. ローソク足の強弱（ロング目線）

| 番号 | ローソク足タイプ | 評価 | エントリー判断 |
|------|----------------|------|--------------|
| ① | 大陽線 | 強い | 逆張り厳禁。連続大陽線は損切り確定 |
| ② | 中陽線 | モブ | エントリー指標にならない |
| ③ | 前回陰線を包む陽線 | 強い | ロング。転換点で多く見られる |
| ④ | 上下ヒゲ中ピンバー陽線 | 弱い | ポジション持ってはいけない。4本以上連続でポジション撤収 |
| ⑤ | 下ヒゲ陽線 | 底打ちっぽい | 想定ロングエントリー位置かつこのローソクで高確率ロング |
| ⑥ | ピンバー強陽線 | 強い | 下ヒゲでキャッチできていないと消耗。次足更新時に陽線なら反転気配 |
| ⑦ | 売り押し目の陽線 | モブ | 下降トレンド中の何の変哲もないモブ。次足更新と同時にショート |

## 3. プライスアクション7種（ロング）

| パターン | 説明 | エントリー |
|---------|------|----------|
| 大陽線 | 強いが逆張り厳禁 | 乗らない |
| 安値圏でのピンバー | そこそこ信用できる反転シグナル。下ヒゲが長いほど安心 | ヒゲ先付近でロング |
| 安値圏での下ヒゲ陽線 | ダブルボトム左肩に多い | ロング |
| リバーサルロー | 最強。ほぼ安値確定。4h・日足で確定させに来る時は適当にロングしても儲かる | ロング |
| 一定価格以下下ヒゲ確定 | 安値圏ピンバーの亜種。実体確定位置が揃う | 確定時間付近でロング |
| 無限スラストアップ | 何があっても無限に安値を切り上げていく | レンジ突入待ち or リバーサルハイ狙いショート |
| 上ヒゲ埋めムーブ | スパイクハイのヒゲを実体で埋めながら高値更新 | 実体レジスタンスで部分利確、ヒゲ埋め完了で全利確 |

## 4. チャートパターン（主要）

- **ダブルボトム**: 2回目のアタックで狼狙いしない。左肩の下ヒゲ陽線がシグナル
- **トリプルボトム**: 今から安値を少し更新するので1度損切りになる可能性あり
- **リバーサルロー**: 安値圏でこの動きが来たときに浅い戻り売りをすると焼ける
- **ミスプライス（ヒゲ）**: ヒゲ先 = ミスプライス。ヒゲを埋めにくるムーブは即ポジフラットまたは持ち替え

## 5. エントリールール（核心）

### ロングエントリー条件
1. 上位足（4h/日足）でレジサポ確認
2. 安値圏に到達
3. 以下のいずれかのプライスアクションが出現：
   - リバーサルロー（最強）
   - 安値圏でのピンバー（下ヒゲ長いほど良い）
   - 安値圏での下ヒゲ陽線（前回陰線が大きいほど良い）
   - 前回陰線を包む陽線

### ショートエントリー条件（ロングの逆）
1. 上位足（4h/日足）でレジサポ確認
2. 高値圏に到達
3. 以下のいずれかのプライスアクションが出現：
   - リバーサルハイ（最強）
   - 高値圏でのピンバー（上ヒゲ長いほど良い）
   - 高値圏での上ヒゲ陰線
   - 前回陽線を包む陰線

## 6. SL（損切り）ルール

- **SLは背が近い場所**: ヒゲ先の少し外側
- ピンバーの場合: ヒゲ先より少し外側
- リバーサルローの場合: 安値のヒゲ先より少し外側
- ヒゲを埋めにくるムーブが出たら即撤退（ミスプライスが消耗している証拠）
- 上位足レジサポを実体で割ったら撤退

## 7. TP（利確）ルール

- **直近高値・安値で利確**（ユーザー指定）
- 上ヒゲ埋めムーブ: 実体レジスタンスで部分利確、ヒゲ埋め完了で全利確
- ロングの場合: 直近スウィングハイ（上位足の高値）で利確
- ショートの場合: 直近スウィングロー（上位足の安値）で利確

## 8. MTF（マルチタイムフレーム）分析

- エントリー時間軸の12倍上位足でレジサポ確認
- 上位足が強いレジサポ = 損切りされる確率が小さい
- 暴騰暴落時に上位足に予め水平線を引いておくことで止まる可能性が高い位置を認識

## バックテスト実装方針

### シグナル生成ロジック（定量化）

**ロングシグナル**:
- 条件A: 直近20本の最安値付近（ATR×2以内）にいる
- 条件B: 以下のいずれか：
  - ピンバー: 下ヒゲ > 実体×2 かつ 実体 > 0（下ヒゲ陽線）
  - リバーサルロー: 前回が大陰線（実体 > ATR×0.8）かつ今回が陽線で前回安値を下回らない
  - 包み足: 今回の実体が前回の実体を完全に包む陽線
- 条件C: 上位足（4h）でサポート付近にいる

**ショートシグナル**:
- 条件A: 直近20本の最高値付近（ATR×2以内）にいる
- 条件B: 以下のいずれか：
  - ピンバー: 上ヒゲ > 実体×2 かつ 実体 > 0（上ヒゲ陰線）
  - リバーサルハイ: 前回が大陽線かつ今回が陰線で前回高値を上回らない
  - 包み足: 今回の実体が前回の実体を完全に包む陰線
- 条件C: 上位足（4h）でレジスタンス付近にいる

**SL**: スウィングロー/ハイのヒゲ先（BacktestEngineの_find_swing_low/highを使用）
**TP**: 直近スウィングハイ/ロー（20本ルックバック）
