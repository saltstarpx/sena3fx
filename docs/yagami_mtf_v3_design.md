# やがみ式3層MTF戦略 (v3) 論理設計書

## 1. 概要
本設計書は、やがみ氏本人から提供された「4時間足（環境認識）→ 15分足（パターン・MTF）→ 1分足（執行）」という3層マルチタイムフレーム（MTF）戦略に基づき、高精度なエントリー・エグジットロジックを実装するための論理構造を定義します。特に、時間軸間の連携と、1分足での精密な執行に焦点を当てます。

## 2. 各時間軸の役割と連携

### 2.1. 4時間足 (H4): 環境認識と「場所」の特定
*   **役割**: 大局的なトレンド方向、主要なサポート/レジスタンスライン、および「実体の揃い」による強力な反転ポイント（壁）を特定します。
*   **ロジック**: 
    *   **トレンド判定**: 長期移動平均線（例: EMA200）の方向や、高値・安値の切り上げ・切り下げでトレンドを認識。
    *   **主要な壁**: 過去の重要な高値・安値、および「実体の揃い」が顕著な価格帯を特定。これらの価格帯が、15分足でのエントリー検討ゾーンとなります。
    *   **4H足形成の予測**: 15分足の動きから、現在の4時間足がどのような形状（例: 下髭陽線、上髭陰線）で確定しそうかを予測します。特に、4時間足の確定まで残り15分〜30分（15分足1〜2本分）の期間に注目します。

### 2.2. 15分足 (M15): パターン認識と「タイミング」の探索
*   **役割**: 4時間足で特定された「場所」において、具体的な反転パターンやエントリーの「タイミング」を探索します。4時間足のローソク足形成を考慮したMTF分析を行います。
*   **ロジック**: 
    *   **実体の揃い**: 15分足レベルでの「実体の揃い」を検知。これは、4時間足の壁付近でのボラティリティ収縮を示唆します。
    *   **インサイドバー**: 15分足でのインサイドバーの連続出現を検知。これは、エネルギー蓄積とブレイクアウトの予兆と捉えます。
    *   **チャートパターン**: 逆三尊、ダブルボトム/トップ、ヘッドアンドショルダーズなどの反転パターンを検知。特に「右肩」の形成に注目します。
    *   **4H足とのMTF**: 4時間足の確定形状を予測し、その予測に合致する15分足パターンが出現した場合に、1分足での執行準備に入ります。

### 2.3. 1分足 (M1): 精密な執行と「背」の確定
*   **役割**: 15分足でエントリーの「タイミング」が到来したと判断された後、1分足の微細な動きを捉え、最小のリスクでエントリーを執行します。ストップロス（SL）を極限までタイトに設定し、リスクリワード比（RR）を最大化します。
*   **ロジック**: 
    *   **微細な収束**: 1分足レベルでの「実体の揃い」やインサイドバーの連続を検知。これは、エントリー直前の最終的なボラティリティ収縮を示します。
    *   **ブレイクアウト**: 収束後の明確なブレイクアウト（実体での高値/安値更新）をエントリートリガーとします。
    *   **タイトなSL**: エントリー直前の1分足の安値（ロングの場合）または高値（ショートの場合）の髭先を「背」として設定し、SLを極限までタイトにします。
    *   **TP設定**: 4時間足の次の壁、または直近の15分足スイングハイ/ローを目標とします。

## 3. ロジックフロー (ロングエントリーの例)

1.  **H4環境認識**: USD/JPYが上昇トレンド中であり、かつ、過去の重要なサポートラインまたは4H足の「実体の揃い」による強力な壁に到達していることを確認。
2.  **M15パターン認識**: 
    *   H4の壁付近で、15分足が「実体の揃い」を形成し、ボラティリティが収縮（インサイドバー連続）していることを確認。
    *   同時に、15分足で逆三尊の右肩が形成されつつある、または4時間足が下髭陽線で確定しそうな状況を予測。
3.  **M1執行判断**: 
    *   M15でエントリータイミングが到来したと判断された後、1分足に切り替えて監視。
    *   1分足でさらに微細な「実体の揃い」またはインサイドバーの連続を確認。
    *   その後の1分足が、実体で直近の高値を明確にブレイクアウトした瞬間にロングエントリー。
    *   エントリー直前の1分足の最安値（髭先）にタイトなSLを設定。
    *   TPはH4の次の壁、またはM15の直近スイングハイに設定。

## 4. 実装上の課題と考慮事項
*   **データ取得**: 1分足データは膨大になるため、効率的なデータ管理と処理が必要です。
*   **時間軸同期**: 異なる時間軸のデータを正確に同期させ、リアルタイムに近い形で分析を行うためのフレームワークが必要です。
*   **パラメータ最適化**: 各時間軸における「実体の揃い」の閾値、インサイドバーの連続回数、SL/TPの設定など、多岐にわたるパラメータの最適化が必要です。
*   **計算負荷**: 1分足データでのバックテストは計算負荷が高いため、効率的なアルゴリズム設計が求められます。
