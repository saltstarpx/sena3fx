## 分析の目的と方法

以下の4シナリオでバックテストを実行し、結果を比較した。

| **D（両方バグ）** | C + B の複合 | BUG-001+002の複合影響 |

## バグ別の影響評価

### BUG-001（27カラムバグ）の影響: **壊滅的**

4時間足の `high`, `low`, `close`, `ema` がすべて `NaN` になるため、v76戦略の上位足フィルター（4時間足EMA・トレンド判定）が完全に機能しなくなる。その結果、**シグナルが1件も生成されず、トレード数は0回**になる。

このバグが存在した状態でバックテストを実行しても「トレードなし」という結果になるため、**バグの存在自体に気づきにくい**という点が特に危険である。実際に `USDJPY_15m.csv` および `USDJPY_4h.csv` を使用していた旧バックテストスクリプト（`run_backtest_htf.py`, `run_backtest_run003.py` 等）は、このバグの影響を受けていた可能性が高い。

> **重要:** 現行の `backtest_full_oanda.py` は `usdjpy_is_4h.csv` / `usdjpy_oos_4h.csv` を使用しており、これらは当初から27カラムバグの影響を受けていない。v60以降のバックテスト結果は正常なデータに基づいている。

### BUG-002（月末タイムスタンプ重複）の影響: **軽微**

月末重複バグが存在する場合、IS+OOS全体でトレード数が **580回 → 601回（+21回）** に増加し、総損益は **+16,921pips → +16,714pips（-207pips、-1.2%）** に低下する。PFは **2.63 → 2.53（-0.10）** と小幅に悪化する。

この差異は、月末の重複行が「追加のシグナル」として誤認識されることで、本来は発生しないはずの21回のトレードが追加されるためである。これらの追加トレードは勝率・PFが若干低く、全体の統計を若干悪化させる。

ただし、**統計的有意性（p値<0.0001）とプラス月数（20/20）は変化しない**。月末重複バグが存在していた状態でのバックテスト結果は、正規データと比べて「やや保守的（悲観的）」な数値になっていたと言える。

### 月別への影響（BUG-002）

月末重複バグの影響は特定の月に集中している。2024年10月と2025年9月に大きなマイナス差分（それぞれ約-250pips、-60pips）が見られる。これらの月は月末にトレンドの転換があり、誤認識された追加シグナルが損失を生んだと考えられる。

## 結論

**修正後の正規データによるバックテスト結果（IS+OOS統合）:**

- トレード数: **580回**（20ヶ月間）
- 勝率: **56.7%**
- PF: **2.63**（IS: 2.98 / OOS: 2.42）
- 総損益: **+16,921pips**
- ケリー基準: **0.352**（推奨ベット比率: 35.2%）
- 統計的有意性: **p < 0.0001**（極めて高い有意性）
- プラス月: **20/20ヶ月（100%）**

月末重複バグは結果を若干悲観的に見せていたに過ぎず、修正後の正規データではPFが **2.53 → 2.63** に改善している。27カラムバグは `usdjpy_is/oos_4h.csv` には存在しなかったため、v60以降のバックテスト結果への影響はない。

**現行のv76戦略の優位性は、データ修正後も完全に維持されている。**
