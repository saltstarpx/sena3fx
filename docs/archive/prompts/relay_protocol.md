# プロンプトリレープロトコル

本ドキュメントは、AIエージェント間（ClaudeとManus）の連携におけるプロンプトの中継ルールを定義します。

## 1. 目的

- AIエージェント間のコミュニケーションを明確化し、誤解を排除する。
- プロンプトの意図、背景、期待される出力を構造化し、効率的な連携を促進する。
- 履歴管理を容易にし、PDCAサイクルを円滑にする。

## 2. 基本原則

- **明確性**: プロンプトは曖昧さを排除し、具体的かつ明確に記述する。
- **構造化**: 定義されたフォーマットに従い、必要な情報を漏れなく記述する。
- **トレーサビリティ**: 全てのプロンプトには一意のRunIDを付与し、関連するLogbook EntryIDと紐付ける。

## 3. プロンプトフォーマット

各プロンプトは以下の要素を含むMarkdownファイルとして `docs/prompts/runs/` ディレクトリに保存されます。

```markdown
# RunID: [YYYYMMDD-HHMMSS-XXX]

- **日付**: [YYYY-MM-DD HH:MM:SS]
- **発議者**: [AIエージェント名またはユーザー名]
- **関連EntryID**: [LogbookのEntryID (例: 20260305-001)]
- **目的**: [このプロンプトの目的を簡潔に記述]
- **背景**: [このプロンプトに至った経緯や現状認識]
- **入力データ**: [使用するデータ、その出所、形式など]
- **期待される出力**: [どのような形式で、どのような情報が欲しいか]
- **制約事項**: [プロンプト実行上の制約や考慮事項]
- **具体的な指示**: [AIエージェントへの具体的なタスク指示]

## 実行結果

[実行結果の概要、または結果ファイルへのリンク]
```

## 4. RunIDの採番ルール

`YYYYMMDD-HHMMSS-XXX` の形式で、日付、時刻、および3桁の連番（XXX）を使用します。

## 5. 運用フロー

1. 新しい検証やPDCAサイクルを開始する際、発議者は `docs/prompts/runs/` に新しいMarkdownファイルを作成し、上記フォーマットに従ってプロンプトを記述する。
2. 関連する仕様変更や意思決定がある場合、`docs/logbook.md` にEntryIDを追記し、プロンプト内の `関連EntryID` にそのIDを記載する。
3. プロンプトを実行したAIエージェントは、実行結果の概要をプロンプトファイル内の `実行結果` セクションに追記するか、結果ファイルへのリンクを記載する。
