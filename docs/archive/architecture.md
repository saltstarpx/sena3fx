# システムアーキテクチャ

## 概要

本プロジェクトは、自律型自動取引エージェントの開発を目的としています。主要なコンポーネントは以下の通りです。

- **データ収集モジュール**: 取引データ（ティックデータ、OHLCバーなど）を収集し、前処理を行います。
- **シグナル生成モジュール**: 収集されたデータに基づき、取引シグナルを生成します。
- **バックテストエンジン**: 生成されたシグナルと過去データを用いて、取引戦略のパフォーマンスを評価します。
- **リスク管理モジュール**: ポジションサイズ、損切り、利食いなどのリスクパラメータを管理します。
- **実行モジュール**: 実際の取引プラットフォームと連携し、シグナルに基づいた注文を実行します。
- **ロギング・モニタリングモジュール**: 取引履歴、パフォーマンス、システム状態などを記録し、監視します。

## データフロー

1. **データソース**: Polygon.ioなどのAPIから生データ（ティックデータ）を取得します。
2. **前処理**: ティックデータからOHLCバーを生成し、各種テクニカル指標を計算します。
3. **シグナル生成**: 前処理されたデータを用いて、取引シグナル（買い/売り/なし）を生成します。
4. **バックテスト**: シグナルと過去データを用いて、戦略の有効性を検証します。
5. **リスク管理**: バックテスト結果に基づき、リスクパラメータを調整します。
6. **リアルタイム実行**: リアルタイムデータに対してシグナルを生成し、リスク管理モジュールを介して取引を実行します。
7. **ログ記録**: 全ての取引、シグナル、パフォーマンスデータはログとして記録されます。
