# sena3fx 戦略開発議事録
## Yagami MTF v60 〜 v76 ロジック変遷記録

**プロジェクト**: sena3fx（やがみ式FX自動売買エージェント）  
**対象通貨ペア**: USDJPY  
**バックテスト期間**: 2024年7月1日 〜 2025年2月6日（約8ヶ月）  
**スプレッド**: 0.2pips（基準）/ 0.4pips（実運用想定）  
**作成日**: 2026年3月6日

---

## 概要

本議事録は、やがみ氏のトレードnoteを参考に構築したFX自動売買戦略「sena3fx」の、v60からv76に至る開発経緯をまとめたものである。各バージョンの変更点・バックテスト結果・廃止理由を時系列で記録し、現行ロジックの根拠を明確化することを目的とする。

やがみ式の核心は「**4時間足で方向感を確認し、15分足のチャートパターンでエントリーポイントを探し、1分足の動きを見て最終エントリーする**」というマルチタイムフレーム（MTF）分析にある。損切りは「チャートパターンが崩れた場所」に置き、RRを2.5〜5.0倍に設定することで、勝率が低くても最終的に利益を出す構造を目指している。

---

## 前史：v58〜v59（参照ベース）

v60以降の理解のために、直前の基盤バージョンを簡記する。

**v58**は本プロジェクトの原型となったバージョンで、4時間足EMA20によるトレンド判定と、15分足の下髭・上髭（ATR×0.5以上）を反発サインとして検出し、髭の50%戻し地点まで引きつけて指値エントリーする方式を採用した。RR 5.0設定で、バックテスト期間（2024年7月〜2025年2月）においてPF 1.85〜1.85程度を記録した。

**v59**はv58にボラティリティ・アダプティブ・フィルター（ATRがスプレッドの3倍未満の低ボラ時にエントリー停止）とタイム・ディケイ（エントリー後15分で含み益が一定水準未満なら強制決済）を追加した。PFは8.11まで向上したが、タイム・ディケイが「伸びるトレード」の利益を制限する問題が指摘された。水原様の指示により、**タイム・ディケイは以降のバージョンで不採用**とすることが決定された。

## v60：上位足フィルターの強化とボラティリティ問題の顕在化

**変更点**: v59ベースに、1時間足EMA20・EMA75と4時間足EMA20・EMA75の「EMAの並び」による厳格なトレンドフィルターを追加。ロングは「1時間足・4時間足ともにclose > EMA20 > EMA75」を必須条件とした。

**バックテスト結果**:

| フィルター設定 | エントリー数 | 勝率 | PF | 総損益 |
| :--- | :---: | :---: | :---: | :---: |
| 有効（閾値3.0倍） | 0回 | — | — | 0pips |
| 無効（閾値0.0倍） | 115回 | 8.7% | 0.046 | -13.9pips |
| 緩和（閾値0.5倍） | 60回 | 16.7% | 0.103 | -5.9pips |

**評価と廃止理由**: ボラティリティフィルターの閾値を3.0倍に設定すると一切エントリーが発生しないという致命的な問題が判明した。緩和しても PF 0.1程度に留まり、EMAの並びによるフィルターが機会損失を過度に生み出していることが確認された。

## v61〜v62：エントリー重複バグの発見と修正

**v61の変更点**: v60のEMAフィルターを維持しつつ、ボラティリティフィルターを0.8倍に調整。デバッグログを追加して問題の特定を試みた。

**v62の変更点**: 動的ボラティリティフィルター（ATR_MA×0.8倍）と時間帯フィルター（UTC 15:00〜21:00 = 日本時間深夜の取引停止）を組み合わせた。

**バックテスト結果（v62）**: エントリー数157回・総損益 -1,525pips・PF 0.27

**発見された根本問題**: 15分足の1シグナルに対し、1分足で条件を満たす限り連続してエントリーが発生する「重複エントリーバグ」が存在していた。157回というエントリー数の異常な多さはこのバグによるものであり、実質的なバックテスト結果として無効であった。

## v68〜v70：やがみnote準拠への本格移行

v60〜v67が「15分足の髭50%戻し指値エントリー」を採用していたのに対し、v68以降はやがみnoteの「更新タイミングでの成行エントリー」に方針転換した。

### v68：タイム・ディケイ廃止・ボラティリティフィルター再設計

**変更点**:
- v58/v59の核心ロジックを継承しつつ、タイム・ディケイを完全廃止（水原様の指示）
- ボラティリティフィルター: ATR < スプレッド×3 の極端な低ボラ時のみ除外
- SL: 髭の先端 + ATR×0.5、TP: リスク×5.0

**バックテスト結果（v68）**: エントリー数は改善されたが、RR 5.0の設定が依然として勝率低下を招いた。

### v69：髭閾値の緩和と1時間足フィルター除外

**変更点**:
- 髭閾値を ATR×0.5 → ATR×0.3 に緩和（エントリー機会の最大のボトルネックを解消）
- 1時間足トレンドフィルターを除外（v58本来のシンプルさに回帰）
- SL: 髭の先端 - ATR×0.5、TP: リスク×5.0

**評価**: 髭閾値の緩和によりエントリー数は増加したが、RR 5.0の設定が依然として課題であった。

### v70：15分足更新タイミング成行エントリーへの転換

**変更点**:
- エントリー方法を「50%戻し待ち指値」から「次の15分足の始値で成行」に変更
- やがみnote引用: 「後1分待ってもし15分での初動が陽線なら反転する確率は上がる」「更新から2分くらいのイメージ」
- 損切り位置: 髭の先端（チャート構造上の意味ある場所）
- RR: 5.0（v58/v59と同一）

**評価**: 成行エントリーへの転換によりスリッページリスクが増加したが、エントリータイミングの明確化という点でやがみnoteの意図に近づいた。

## v73：4時間足更新タイミング＋二番底・二番天井ロジックへの大転換

v68〜v70の「15分足髭反発」ロジックから、**「4時間足の二番底・二番天井」**という全く異なるパターン認識ロジックへの大転換を行った。これはやがみnoteの「4時間足のローソク足形成を考慮する」という記述に基づく。

## v76：スプレッド計算の実環境対応（バグ修正）

**発見された問題（v75の設計バグ）**:

v75ではスプレッドをエントリー価格に加算した後、そのEPを基準にリスク幅（risk）を計算していたため、スプレッドがriskに吸収される構造になっていた。

```
v75の問題:
  EP = 始値 + spread
  risk = EP - SL  ← スプレッドがriskに含まれる
  TP = EP + risk × RR  ← TPもスプレッド分だけ遠くなる
  → スプレッドを0.2→0.4に変えても損益がほぼ変わらない
```

**変更点（v76）**:

SL/TPをチャートレベル（始値基準）で固定し、スプレッドをエントリーコストとして損益に正しく反映する実装に修正した。

```
v76の正しい実装:
  raw_ep = 始値（チャートレベルの基準価格）
  EP = raw_ep + spread（実際の約定価格）
  risk = raw_ep - SL  ← チャートレベルで計算
  TP = raw_ep + risk × RR  ← チャートレベルで計算
  損益 = (exit_price - EP) × 100 × dir
    → SL到達時: -(risk + spread) × 100
    → TP到達時: (risk × RR - spread) × 100
```

**バックテスト結果（v76、スプレッド0.2pips）**:

| 指標 | 値 |
| :--- | :---: |
| エントリー数 | 53回 |
| 勝率 | 62.3% |
| PF | 4.71 |
| 総損益 | +2,693pips |

**スプレッド感度（v76）**:

| スプレッド | 総損益 | PF |
| :---: | :---: | :---: |
| 0.2pips | +2,693pips | 4.71 |
| **0.4pips** | **+2,687pips** | **4.68** |

スプレッドが0.2→0.4pipsに倍増しても総損益の低下は約6pips（53トレード×約0.1pips/トレード）に留まり、**実運用環境（スプレッド0.4pips）でも十分な優位性を維持**することが確認された。

**損切り幅感度分析（v76、スプレッド0.4pips）**:

| SL設定 | エントリー数 | 総損益 | PF | 全月プラス |
| :---: | :---: | :---: | :---: | :---: |
| ATR×0.10 | 47回 | +2,474pips | 4.95 | 8/8ヶ月 |
| **ATR×0.15（現行）** | **53回** | **+2,687pips** | **4.68** | **8/8ヶ月** |
| ATR×0.25 | 46回 | +2,604pips | 4.67 | 8/8ヶ月 |
| ATR×0.35 | 40回 | +2,299pips | 4.21 | 7/8ヶ月 |

**評価**: ATR×0.15（現行設定）が総損益・エントリー数・全月プラスのバランスで最優秀。ATR×0.35はプラス月数が7/8に低下するため非推奨。**現行のATR×0.15を採用することを決定**した。

## 主要な学習事項

**1. 重複エントリーバグの危険性**  
v62で発見されたように、1つのシグナルに対して複数回エントリーが発生するバグは、バックテスト結果を根本的に歪める。シグナル生成ロジックには必ず「used_times」等の重複防止機構が必要である。

**2. エントリー数とサンプルの重要性**  
v63・v64のようにPFが高くてもエントリー数が4回では統計的有意性がない。t検定でp値 < 0.05を達成するには最低30〜50回以上のサンプルが必要であり、v74以降の53回でようやく実用水準に達した。

**3. RR比率の最適値**  
v67の検証でRR 5.0が非効率であることが判明。v75の定量分析でRR 2.5が最適であることを確認した。高RRを狙いすぎると利確が遠くなりすぎて勝率が低下し、期待値が下がる。

**4. スプレッドの正しい計算方法**  
v75で発見されたスプレッド計算バグは見落としやすい。SL/TPはチャートレベル（始値基準）で固定し、スプレッドはエントリーコストとして損益に別途反映する必要がある。

**5. 二番底・二番天井ロジックの優位性**  
v68〜v70の「髭反発」ロジックから、v73以降の「二番底・二番天井」ロジックに転換したことで、PFが劇的に改善した。チャートパターンの明確な定義（前々回安値≈前回安値±ATR×0.3）が過学習を防ぎつつ高い優位性を実現している。
