# 「1分足スキャルピング的執行」におけるレイテンシとスリッページのリスク評価

やがみ式v58ロジックの「1分足の髭の先端を捉え、即含み益で伸ばす」という特性を最大限に活かすためには、注文の**レイテンシ（遅延）**と**スリッページ（約定価格のずれ）**を最小限に抑えることが不可欠です。ここでは、OANDA v20 APIとGMOコイン FX APIの特性を踏まえ、それぞれのプラットフォームにおけるリスクを評価します。

## 1. レイテンシ（注文遅延）のリスク評価

レイテンシは、シグナル発生から注文が取引所に到達し、処理されるまでの時間差を指します。1分足の髭の先端を狙う戦略では、この時間差が致命的な損失に繋がりかねません。

### OANDA v20 API (東京サーバー)
*   **技術的優位性:** OANDAは世界中のアルゴトレーダーに利用されており、APIの設計自体が低レイテンシを意識しています。特に東京サーバー（TY3）は、日本のトレーダーにとって物理的な距離が短く、ネットワークレイテンシを最小限に抑えることが可能です。
*   **注文制限:** 秒間120回という高いリクエスト制限は、高頻度な注文発注や、シグナル発生時の複数注文（例：エントリーと同時に損切り・利確注文）をスムーズに行う上で大きなメリットとなります。
*   **リスク:** サーバーの混雑状況やネットワーク環境に依存しますが、適切なVPS（Virtual Private Server）を東京近郊に設置することで、極めて低いレイテンシを実現できる可能性が高いです。

### GMOコイン FX API
*   **技術的特性:** 国内の取引所としてはモダンなAPIを提供していますが、POST注文（新規注文や変更）の制限が「秒間1回」と厳しく設定されています。
*   **注文制限:** この「秒間1回」という制限は、1分足の非常に短い時間枠で複数のシグナルが発生したり、高速なポジション調整が必要になったりする場合に、注文の遅延や失敗を引き起こす大きなリスクとなります。
*   **リスク:** 1分足の髭の先端を捉えるためには、シグナル発生から数秒以内の注文執行が求められますが、この制限により機会損失や不利な約定価格での執行リスクが高まります。

## 2. スリッページ（約定価格のずれ）のリスク評価

スリッページは、注文価格と実際の約定価格との間に発生する差です。特に相場が急変動する局面や、流動性が低い時間帯に発生しやすくなります。やがみ式v58ロジックは、損切り幅を小さく設定しているため、わずかなスリッページでもリスクリワード比率に大きな影響を与えます。

### OANDA v20 API
*   **約定力:** OANDAはNDD（No Dealing Desk）方式を採用しており、顧客の注文を直接市場に流すため、透明性が高いとされています。また、流動性プロバイダーが複数いるため、比較的安定した約定が期待できます。
*   **スリッページ対策:** APIを通じて「許容スリッページ」を設定できる機能がある場合、不利な約定を避けることができます。ただし、その場合は約定しないリスクも伴います。
*   **リスク:** 極端な相場変動時にはスリッページが発生する可能性はありますが、GMOコインと比較してそのリスクは低いと考えられます。

### GMOコイン FX API
*   **約定力:** 国内取引所の多くはDD（Dealing Desk）方式を採用しており、GMOコインもカバー先との関係で約定価格が決定されます。急変動時にはスプレッドが拡大したり、スリッページが発生しやすくなる傾向があります。
*   **スリッページ対策:** GMOコインのドキュメントにもスリッページに関する注意喚起があり、成行注文や逆指値注文で発生する可能性が明記されています。許容スリッページの設定機能があるかはAPIドキュメントからは明確ではありませんでした。
*   **リスク:** 「1分足の髭の先端」という、まさに相場が急変動している可能性のあるポイントを狙うため、スリッページによる不利な約定リスクがOANDAよりも高いと評価されます。

## 3. 総合的なリスク評価とプラットフォームの選択

やがみ式v58ロジックの「1分足キャッチからの即含み益」という特性を考慮すると、**レイテンシとスリッページのリスクを最小限に抑えることが最重要課題**となります。

| 項目 | OANDA v20 API | GMOコイン FX API |
| :--- | :---: | :---: |
| **レイテンシリスク** | **低** (高スループット、物理的近接性) | **高** (秒間1回の注文制限) |
| **スリッページリスク** | **低** (NDD、複数LP、透明性) | **中〜高** (DD、急変時リスク) |

**結論:**
総合的に判断すると、**OANDA v20 API (東京サーバー)** が、やがみ式v58ロジックの「1分足スキャルピング的執行」において、レイテンシとスリッページのリスクを最も低く抑え、高い期待値を維持できる可能性が高いと評価されます。

GMOコイン FX APIは、国内取引所としての安心感や日本語ドキュメントの分かりやすさというメリットはありますが、高頻度・高精度なスキャルピング戦略においては、その注文制限と約定力のリスクがパフォーマンスに悪影響を及ぼす可能性が高いです。
