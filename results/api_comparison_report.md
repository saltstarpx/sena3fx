# APIプラットフォーム比較報告書: OANDA v20 vs GMOコイン FX

「1分足の髭の先端を捉え、即含み益で伸ばす」やがみ式v58ロジックに最適な取引環境を特定するための比較調査結果です。

## 1. スペック比較表

| 比較項目 | OANDA v20 API (東京サーバー) | GMOコイン FX API |
| :--- | :--- | :--- |
| **API形式** | REST (v20) / WebSocket / FIX | REST / WebSocket |
| **注文レート制限** | **120回 / 秒** (非常に高い) | **1回 / 秒** (POST注文) |
| **レート取得制限** | ストリーミング (WebSocket) 無制限 | 6回 / 秒 (GET) |
| **レイテンシ** | **極めて低い** (TY3近接VPS推奨) | 標準的 (国内サーバー) |
| **スプレッド (USDJPY)** | 0.3 〜 0.4 pips (安定) | 0.3 pips 前後 (変動あり) |
| **取引手数料** | 無料 (スプレッドのみ) | **外付け手数料あり** (API経由注文) |
| **最小注文数量** | 1通貨単位 | 1,000通貨単位 (一部通貨で引き下げ) |
| **ドキュメント** | 非常に充実 (世界標準) | 充実 (日本語) |

## 2. 詳細分析

### 注文レスポンスと制限
*   **OANDA:** 秒間120リクエストという圧倒的なスループットを誇ります。1分足の髭の先端を狙う際、ミリ秒単位の攻防において制限に掛かるリスクがほぼゼロです。
*   **GMOコイン:** POST注文が「秒間1回」に制限されています。複数のポジションを同時に操作したり、高速な損切り・ドテン注文を行う際にボトルネックになる可能性があります。

### 約定力とスリッページ
*   **OANDA:** 東京サーバー (TY3) は世界中のプロが利用しており、約定の透明性が高いです。特に「指値」に近い成行（FOK/IOC）の挙動が安定しています。
*   **GMOコイン:** 国内取引所特有の「カバー先」への依存があり、急変時のスリッページが発生しやすい傾向があります。また、API経由の注文には別途手数料が発生する点に注意が必要です。

### 開発の容易性
*   **OANDA:** Pythonライブラリ (`oandapyV20`) が成熟しており、バックテストから実運用への移行がスムーズです。
*   **GMOコイン:** 日本語ドキュメントは分かりやすいですが、サードパーティ製ライブラリは少なく、自前での実装箇所が多くなります。

## 3. 総合評価と推奨

| 戦略タイプ | 推奨プラットフォーム | 理由 |
| :--- | :--- | :--- |
| **短期・高頻度 (v58)** | **OANDA v20 (東京)** | **秒間120回の注文制限と低レイテンシ**が、髭の先端を捉える「精度」を支えるため。 |
| **中長期・低頻度** | GMOコイン | 国内の安心感とシンプルなAPI。ただし秒間1回の制限が壁。 |

### 結論
やがみ式v58ロジックの「即含み益で伸ばす」という特性を最大限に活かすには、**OANDA v20 API (東京サーバー)** が最適です。GMOコインはスプレッドこそ狭いものの、APIの注文制限（秒間1回）と外付け手数料が、高精度なスキャルピング執行において足かせとなるリスクが高いと判断しました。
