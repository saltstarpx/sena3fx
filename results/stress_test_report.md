# ストレステスト＆改善報告書

## 1. 重大な発見: バックテストエンジンのスリッページ未適用バグ

**本検証の最大の収穫は「PF 3.137」の疑義ではなく、エンジン自体のバグ発見である。**

### 問題
`lib/backtest.py` の `slippage_pips` パラメータは `self.slippage` に格納されていたが、**PnL計算に一切使用されていなかった**。つまり、これまでの全バックテスト結果（RUN-001～010）は**スプレッド・手数料ゼロの理想的環境**で計算されていた。

### 修正
PnL計算にスリッページを適用:
```python
# 修正前
pnl = (exit_price - l['entry']) * l['size']

# 修正後 — 往復スリッページを差し引く
pnl = (exit_price - l['entry']) * l['size']
pnl -= 2 * self.slippage * l['size']
```

## 3. 生存者バイアス分析 (前半/後半分割)

### 評価
- **PA v2**: 前半/後半ともにPF>1.5。最も安定。

- **DC50+EMA200**: 後半の金上昇トレンドに大きく依存。前半PF=1.52は許容範囲だが、2021-2023のレンジ相場では機能しない（年別データ参照）。

## 5. 総合評価 — 実戦投入可否判定

| 項目 | PA v2 | MTF v3 | DC50+EMA200 |
|------|:---:|:---:|:---:|
| スプレッド耐性 | A (1.0pips: PF=1.88) | C (1.0pips: PF=1.14) | A (1.0pips: PF=2.85) |
| 前半/後半安定性 | A (両方PF>1.5) | D (前半PF=1.07) | B (前半PF=1.52) |
| 年別安定性 | B (2024-2025合格) | D (2/4年不合格) | D (3/7年壊滅) |
| 1トレード依存度 | B (N=53, 分散) | A (N=203, 高分散) | D (N=43, 年6回) |
| **総合** | **A-** | **C+** | **B-** |

### 結論

1. **PA v2 (推奨度: ★★★★☆)**: スプレッド耐性・前後半安定性ともに最強。N=53がやや少ないが、4H足のため1回あたりの利益幅が大きくスプレッドの影響を吸収。**実戦デモ投入に最も近い戦略。**

2. **DC50+EMA200 (推奨度: ★★★☆☆)**: スプレッド耐性は優秀だが、**2021-2023の3年連続壊滅**が致命的。金のトレンドが反転した場合に大きなドローダウンを被る。レンジ相場フィルターの追加が必須。

3. **MTF v3 (推奨度: ★★☆☆☆)**: スプレッド耐性が弱く、年別でも不安定。前半PF=1.07はエッジが存在しないに等しい。**現状では実戦投入不可。** 改善策としてカスタムSL（ヒゲ先SL）の実装が必要。
