# CLAUDE.md

sena3fx — 自律型自動取引エージェント

## プロジェクト概要

やがみメソッドに基づくMTF（マルチタイムフレーム）トレンドフォロー戦略の自動化システム。
Cloud Run上でペーパートレードを24時間稼働させ、PDCAサイクルで継続的に戦略を改善する。

- **現在の戦略バージョン**: **F1+F3 トレンドフォロー型**（2026/3/9 確定）
- **採用銘柄**: EURUSD / GBPUSD / AUDUSD / USDCHF / EURGBP
- **データソース**: OANDA API（リアルタイム）/ ローカルCSV（バックテスト用）
- **ブランチ**: `main`
- **リモート**: GitHub `saltstarpx/sena3fx`

## 現行戦略: F1+F3 トレンドフォロー型

### 概要

4時間足EMA20によるトレンドフィルター + 1時間足/4時間足の緩いパターン検出（旧ロジック）。
**F1（時間帯フィルター）+ F3（1H足方向確認）** を適用した確定版。

エントリーは足更新後2分以内の1分足始値（成行）。半利確あり（1R到達でポジション半分決済・SLをBEへ）。

### エントリー条件（全条件AND）

| 条件 | 内容 |
|------|------|
| トレンド一致 | 4H EMA20より上→ロングのみ、下→ショートのみ |
| パターン成立 | 前2本の安値/高値差が ATR×0.3 以内 + 直近足が方向一致の実体 |
| KMID | 直前足の実体方向がエントリー方向と一致 |
| KLOW | 直前足の下ヒゲ比率 < 0.15% |
| **F1: 時間帯** | **UTC 5〜15時のみ（ロンドン〜NYオーバーラップ）** |
| **F3: 1H確認** | **1H足のトレンド方向が4H足と一致** |
| リスク幅 | SLまでの距離が ATR×3（4H）または ATR×2（1H）以内 |

### SL/TP設定

- **SL**: パターンの安値・高値から ATR×0.15 外側
- **TP**: SLリスク幅 × 2.5倍
- **半利確**: 1R到達でポジション半分決済 → SLをBEへ移動

## 戦略開発の経緯（2026/3/9 セッション）

### 1. バグ修正: シグナル過剰発生の解消

**問題**: 旧バックテスト（v77_4m_lite）でシグナルが月120件/銘柄発生し、リターンが+2000%超という非現実的な数字が出ていた。

**原因**: 二番底パターンの検出条件が「前2本の安値差 ≤ ATR×0.3 + 直近足が陽線」だけで、上昇トレンド中はほぼ毎時間成立してしまっていた。

**修正版（v77修正版）**: 3点構造（底1→山→底2）での厳密な検出に変更。月25件程度、固定ロットPF1.7〜2.0。

### 2. 複利膨張の検証

**疑問**: PF3超は複利水増しではないか？

**結論**: 複利版PFが固定ロット版より1.19倍高い程度で軽微。主因は**半利確後のBE-SLがプラスPnLを大量生成**する構造（SLと表示されているトレードの44%がプラスPnL）。

### 3. 旧ロジックのトレンドフォロー戦略としての再評価

**発見**: 緩い検出条件（旧ロジック）は「二番底パターン」ではなく「EMA20方向へのトレンドフォロー」として機能しており、固定ロットでPF2.0超という優位性があった。

**比較結果（固定ロット2%）**:

| 戦略 | PF | MDD | 月次+ |
|------|-----|-----|-------|
| v77修正版（3点構造） | 1.7〜2.0 | 8〜23% | 4/4 |
| **旧ロジック（トレンドフォロー）** | **2.0〜2.3** | **7〜16%** | **20/20** |

### 4. 同時ポジション上限の最適化

4銘柄（AUDUSD/USDCHF/USDCAD/EURJPY）で同時ポジション上限を比較：

| 上限 | PF | MDD | 最終資産 |
|------|-----|-----|---------|
| 制限なし | 2.09 | 12.1% | 2,212万円 |
| 固定10件 | 2.04 | 9.0% | 803万円 |
| **動的（10→15→20）** | **2.03** | **9.0%** | **1,874万円** |
| 固定20件 | 2.09 | 12.1% | 2,175万円 |

→ **動的（10→15→20）を採用**（MDD固定10件と同水準でリターンは+28%）

### 6. 定量分析による特徴量発見

**分析対象**: ◎採用5銘柄の全取引データ（11,070件）

**発見した特徴量**:

| 特徴量 | 内容 | 効果 |
|--------|------|------|
| **F1: 時間帯フィルター** | UTC 5〜15時のみ（20-21時は勝率17-18%で最弱） | SR +1.17 |
| F2: ストリーク | 前回負け後スキップ | SR +0.74（過学習リスクあり→**除外**） |
| **F3: 1H足優先** | 1H足のトレンド方向が4H足と一致 | SR +0.06 |

**F2除外の根拠**: スキップされた1,216件の勝率29.4% = 全体平均29.2%と同値。過学習リスクに見合わない。

**最終採用**: F1+F3

## utils/risk_manager.py の仕様

```python
rm = RiskManager(pair, risk_pct=0.02)

# ロット計算
lot = rm.calc_lot(capital, risk_distance, ref_price=ep, usdjpy_rate=150.0)

# PnL計算（引数順序に注意）
pnl = rm.calc_pnl_jpy(direction, entry_price, exit_price, lot, usdjpy_rate=150.0, ref_price=ep)
```

`quote_type == "A"` の銘柄（EURUSD等）は `usdjpy_rate` 不要。
`quote_type == "B"` の銘柄（USDJPY等）は `usdjpy_rate` が必要。
